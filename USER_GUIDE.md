## Getting Started

### What is Maia?

Maia is your all-in-one Student Support Services platform. It replaces Excel spreadsheets, Google Forms, and scattered Word documents with a unified system for managing student cases, interventions, and progress tracking.

**Named After:** The Pleiades star Maia (representing guidance and care) and Wendy's daughter Maia

**What Maia Does:**
- Tracks student support cases
- Documents interventions and sessions
- Stores evidence and files
- Manages parent meetings
- Tracks behavior incidents
- Guides you through protocols (Bullying, Child Protection)
- Provides analytics and reports

### Who Can Use Maia?

**SSS Staff (Full Access):**
- Wendy Aragon
- Lindsey
- Jonica

**Teachers (Limited Access):**
- Can view cases for their own students
- Can submit referrals (KID Talk)

**Principals (Read-Only Access):**
- Can view all cases and data
- Can see analytics and reports

**Parents (Future):**
- Not available in current version

### System Requirements

**Computer:**
- Windows, Mac, or Chromebook
- Modern web browser (Chrome, Firefox, Edge, Safari)
- Internet connection (offline mode coming in Week 8)

**Mobile:**
- iOS or Android device
- Mobile-optimized views available
- Read-only access on mobile (full editing on desktop)

## Managing Cases

### What is a Case?

A case represents a student receiving support services. Each case has:
- Student information
- Case type (Academic, SEL, Bullying, etc.)
- Tier level (1, 2, or 3)
- Status (Open, On Hold, Closed, Referred Out)
- Case manager (who's responsible)
- Interventions and sessions

### Creating a New Case

**Step 1: Click "Create New Case"**
From the dashboard or Cases page.

**Step 2: Select Student**
- Search for the student by name
- Or create a new student if they're not in the system

**Step 3: Choose Case Type**
Select from 7 types:
1. **Academic Support** - Reading, math, literacy help
2. **SEL** - Socio-emotional learning (feelings, social skills)
3. **Distinctions** - Special programs/curricula
4. **Conflict Resolution** - Mediation between students
5. **Bullying** - Anti-bullying protocol (complex)
6. **Child Protection** - Safeguarding concerns (highest priority)
7. **Urgent** - Crisis/immediate danger

**Step 4: Set Tier Level**
- **Tier 1:** Classroom strategies (teacher-led)
- **Tier 2:** Targeted interventions (SSS-led)
- **Tier 3:** Individual learning plan (intensive)

**Step 5: Mark as Urgent (if needed)**
Check "Urgent" for:
- Safeguarding concerns
- Child Protection cases
- Highly dysregulated students
- Immediate danger situations

**Step 6: Assign Case Manager**
- Select yourself or another SSS team member
- System will suggest based on workload

**Step 7: Add Description**
Write what support the student needs and why.

**Step 8: Click "Create Case"**

Done! Your case is created and appears in the cases list.

### Viewing All Cases

1. **Go to:** Cases page (from main menu)
2. **See:** List of all cases (if you're SSS staff)
3. **Filter by:**
   - Status (Open, On Hold, Closed)
   - Case Type
   - Tier Level
   - Urgent flag
   - Case Manager
   - Student Grade

4. **Sort by:**
   - Date created (newest first)
   - Student name (A-Z)
   - Priority (urgent first)

### Viewing a Single Case

1. **Click** any case from the list
2. **See:**
   - Student details
   - Case information
   - All interventions
   - All sessions
   - Files attached
   - History of changes

### Editing a Case

1. **Open** the case
2. **Click** "Edit Case"
3. **Update:**
   - Status (Open, On Hold, Closed)
   - Tier level (if student escalates or de-escalates)
   - Urgent flag
   - Description
   - Case manager

4. **Click** "Save Changes"

### Closing a Case

When a student no longer needs support:

1. **Open** the case
2. **Click** "Close Case"
3. **Select** closure reason:
   - Student showed improvement
   - Referred to external services
   - Family moved/transferred schools
   - Other (explain in notes)

4. **Add** closure notes
5. **Click** "Confirm Closure"

The case moves to "Closed" status but remains in the system for record-keeping.

### Putting a Case On Hold

If you need to pause a case temporarily:

1. **Edit** the case
2. **Change status** to "On Hold"
3. **Add note** explaining why (e.g., "Student on extended medical leave")

### Reopening a Closed Case

1. **Find** the closed case (filter by Status: Closed)
2. **Open** the case
3. **Click** "Reopen Case"
4. **Add note** explaining why
5. **Confirm**

## Tracking Interventions

### What is an Intervention?

An intervention is a specific support strategy or program for a student. Examples:
- Reading comprehension program (Academic)
- Social skills group (SEL)
- Advanced math curriculum (Distinctions)

Each intervention has multiple sessions documenting progress over time.

### Creating an Intervention

1. **Open** a case
2. **Click** "Add Intervention"
3. **Select** intervention type:
   - Academic (reading, math, literacy)
   - SEL (socio-emotional learning)
   - Distinctions (special programs)

4. **Enter** intervention name:
   - Example: "Tier 2 Reading Comprehension"
   - Example: "Feelings Group"

5. **Add** description:
   - What strategies will you use?
   - What are the goals?

6. **Set** schedule:
   - Start date
   - Frequency (e.g., "3x per week")
   - Duration per session (e.g., "30 minutes")

7. **Click** "Create Intervention"

### Viewing All Interventions

1. **Open** a case
2. **Scroll** to "Interventions" section
3. **See:** All interventions for this student

### Editing an Intervention

1. **Click** the intervention
2. **Click** "Edit"
3. **Update** details
4. **Save** changes

### Ending an Intervention

When an intervention is complete:

1. **Edit** the intervention
2. **Set** end date
3. **Change status** to "Completed"
4. **Add** outcome notes:
   - Was it effective?
   - Did the student improve?
   - What's next?

## Documenting Sessions

### What is a Session?

A session is a single meeting or activity with the student. You document what happened, how the student did, and any challenges.

### Creating a Session

1. **Open** an intervention
2. **Click** "Add Session"
3. **Enter** session details:

**Session Date:** When did this session happen?

**Attendance:**
- Present
- Absent
- Excused

**Session Notes:**
Write what happened:
- What did you work on?
- How did the student engage?
- Any breakthroughs or challenges?

Example: "Today we worked on inference skills using picture books. John participated well and made good connections between images and text."

**Progress Rating (1-5):**
- 1 = No progress
- 2 = Minimal progress
- 3 = Some progress
- 4 = Good progress
- 5 = Excellent progress

**Challenges:**
Any difficulties or concerns?
Example: "Still struggles with vocabulary. May need additional support."

4. **Upload** files (optional):
   - Photos of student work
   - Evidence of progress

5. **Click** "Save Session"

### Viewing Session History

1. **Open** an intervention
2. **Scroll** to "Sessions" section
3. **See:** All sessions listed chronologically
4. **Click** any session to view full notes

### Editing a Session

1. **Open** the session
2. **Click** "Edit"
3. **Update** notes or rating
4. **Save** changes

## Uploading Files

### What Files Can You Upload?

**Supported Types:**
- PDF documents
- Word documents (.docx, .doc)
- Photos (.jpg, .png)

**Max File Size:** 10MB per file

**Examples of Files to Upload:**
- Assessment results
- Student work samples
- Photos of evidence
- Intervention plans
- Evaluation reports (secure!)

### How to Upload a File

**Method 1: From a Case**
1. Open the case
2. Click "Upload File" button
3. Select file from your computer
4. File appears in "Files" section

**Method 2: From a Session**
1. Open a session
2. Click "Attach File"
3. Select file
4. File linked to that session

**Method 3: From an Intervention**
1. Open an intervention
2. Click "Upload Evidence"
3. Select file
4. File linked to the intervention

### Viewing Files

1. **Open** the case/intervention/session
2. **Scroll** to "Files" section
3. **Click** file name to download/view

### Deleting a File

1. **Find** the file
2. **Click** trash icon
3. **Confirm** deletion

Warning: Deleted files cannot be recovered!

## FAQ

### General Questions

**Q: Who can access Maia?**
A: SSS staff, teachers, and principals. Each role has different permission levels.

**Q: Is Maia secure?**
A: Yes! We use bank-level encryption, secure Google authentication, and strict access controls. All data complies with GDPR and Spanish education laws.

**Q: Can I access Maia from home?**
A: Yes, as long as you have internet and log in with your ATLAS Google account.

**Q: Does Maia work on mobile?**
A: Yes! The interface is mobile-optimized. Full editing works best on desktop/laptop.

**Q: What happens if the internet goes down?**
A: Offline mode is coming in Week 8. For now, you need internet to use Maia.

### Cases & Students

**Q: How do I know if a case is urgent?**
A: Urgent cases are highlighted in red and appear at the top of lists.

**Q: Can I assign a case to multiple people?**
A: No, each case has one primary case manager. However, multiple people can add notes and sessions.

**Q: What if I made a mistake creating a case?**
A: You can edit the case anytime. Click "Edit Case" and update the information.

**Q: Can I delete a case?**
A: You can archive a case by closing it. Cases are never permanently deleted for audit trail purposes.

**Q: How do I move a student from Tier 1 to Tier 2?**
A: Edit the case and change the Tier level from 1 to 2. Add a note explaining why.

### Interventions & Sessions

**Q: How many interventions can a case have?**
A: As many as needed! A student might have both a reading intervention and a feelings group.

**Q: What if I forget to document a session?**
A: You can add it later. Just enter the correct session date when you create it.

**Q: Can I edit a session after I save it?**
A: Yes! Open the session and click "Edit."

**Q: How do I mark an intervention as completed?**
A: Edit the intervention, set an end date, and change the status to "Completed."

### Files & Evidence

**Q: What file types can I upload?**
A: PDF, Word (.docx), and images (.jpg, .png). Max 10MB per file.

**Q: Where are files stored?**
A: Securely in Supabase Storage (cloud storage). Only authorized users can access them.

**Q: Can I upload photos from my phone?**
A: Yes! If you're logged into Maia on your phone, you can upload photos directly.

**Q: What if I upload the wrong file?**
A: Delete it and upload the correct one. Click the trash icon next to the file.

### Access & Permissions

**Q: Why can't I see all cases?**
A: Teachers can only see cases for their own students. SSS staff and principals can see all cases.

**Q: Why can't I edit a case?**
A: Only SSS staff can create and edit cases. Teachers and principals have read-only access.

**Q: How do I get access to Maia?**
A: Your IT administrator creates your account and assigns your role (SSS Staff, Teacher, or Principal).

## Troubleshooting

### Login Issues

**Problem: "Sign in with Google" button doesn't work**
- Solution: Try a different browser (Chrome recommended)
- Check that pop-ups are not blocked
- Clear browser cache and try again

**Problem: "You don't have permission to access this app"**
- Solution: Contact your IT administrator
- You may not have been added to the Maia user list
- Make sure you're using your ATLAS school email

**Problem: "Session expired, please log in again"**
- Solution: Your session timed out for security
- Click "Log In" again
- You won't lose any saved data

### Page/Display Issues

**Problem: Page won't load**
- Solution: Refresh the page (F5 or Cmd+R)
- Check your internet connection
- Try a different browser

**Problem: Dashboard is blank**
- Solution: You may not have any cases assigned yet
- Refresh the page
- Contact SSS team if you should have access

**Problem: Can't see search results**
- Solution: Clear your filters
- Check spelling in search box
- Try searching by last name only

### File Upload Issues

**Problem: "File too large"**
- Solution: Your file exceeds 10MB
- Compress the file or split into multiple files
- For large files, store in Google Drive and add link in notes

**Problem: "File type not supported"**
- Solution: Only PDF, DOCX, JPG, PNG are allowed
- Convert your file to one of these formats
- For other files, use Google Drive link

**Problem: Upload fails/freezes**
- Solution: Check internet connection
- Try a smaller file
- Refresh page and try again

### Data Issues

**Problem: Changes aren't saving**
- Solution: Check internet connection
- Make sure you clicked "Save" button
- Refresh page and try again (data may have saved)

**Problem: Can't find a student**
- Solution: Check spelling
- Try searching by last name only
- Student may not be in system yet (add them)

**Problem: Case appears in wrong status**
- Solution: Check filters (you may be filtering it out)
- Someone else may have changed the status
- Contact SSS team if this is an error

## Glossary

**Case** - A record of a student receiving support services

**Case Manager** - The SSS staff member responsible for a case

**Case Type** - Category of support (Academic, SEL, Bullying, etc.)

**Intervention** - A specific support strategy or program

**Session** - A single meeting or activity with a student

**Tier 1** - Classroom strategies (teacher-led)

**Tier 2** - Targeted interventions (SSS-led small groups)

**Tier 3** - Intensive individual support

**Urgent** - High-priority cases (safeguarding, crisis, highly dysregulated)

**RLS (Row Level Security)** - Database security that controls who can see what data

**SSS** - Student Support Services (Wendy, Lindsey, Jonica)

**SEL** - Socio-Emotional Learning (feelings, social skills)

**KID Talk** - Referral form teachers use to request SSS support

**Restorative Process** - Follow-up after behavior incidents

**Action Plan** - Follow-up tasks after parent meetings

**Protocol** - Required steps for Bullying or Child Protection cases

## Tips for Success

1. **Document as you go** - Don't wait until Friday to add session notes
2. **Use filters** - Save time by filtering to exactly what you need
3. **Upload evidence** - Photos and files build a stronger case history
